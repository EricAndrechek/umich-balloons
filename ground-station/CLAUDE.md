# Ground Station — Agent Guide

## What This Is

Single Go binary (`umbgs`) running on a Raspberry Pi that receives APRS (via RTL-SDR + Direwolf) and LoRa balloon telemetry and uploads it to an API. Includes an embedded web dashboard on `:8080` with live stats, logs, config editor, and admin tools.

See `README.md` for full architecture, building, and deployment docs.

## Building

```bash
cd ground-station/umbgs
GOOS=linux GOARCH=arm64 CGO_ENABLED=0 go build -o /tmp/umbgs-new ./cmd/umbgs/
```

No CGo. Static binary. Cross-compile from Mac is the normal workflow.

## Deploying to the Live Pi

The test Pi is at `192.168.0.207`, user `umbgs`, password `umbgs`.

**The systemd service runs `/data/umbgs`** — NOT `/usr/local/bin/umbgs`. Always deploy to `/data/umbgs`.

```bash
# 1. Copy binary
sshpass -p 'umbgs' scp -o StrictHostKeyChecking=no -o UserKnownHostsFile=/dev/null /tmp/umbgs-new umbgs@192.168.0.207:/tmp/umbgs-new

# 2. Stop service, replace binary, restart
sshpass -p 'umbgs' ssh -o StrictHostKeyChecking=no -o UserKnownHostsFile=/dev/null umbgs@192.168.0.207 \
  'echo umbgs | sudo -S systemctl stop umbgs'
# Wait a moment for process to die, then:
sshpass -p 'umbgs' ssh ... 'echo umbgs | sudo -S cp /tmp/umbgs-new /data/umbgs && echo umbgs | sudo -S chmod +x /data/umbgs && echo OK'
sshpass -p 'umbgs' ssh ... 'echo umbgs | sudo -S systemctl start umbgs && systemctl is-active umbgs'
```

If `cp` fails with "Text file busy", the old process is still running. `killall -9 umbgs` then retry.

**Always disable strict host key checking** for Pi SSH/SCP (`-o StrictHostKeyChecking=no -o UserKnownHostsFile=/dev/null`).

`sshpass` is available via `brew install hudochenkov/sshpass/sshpass`.

## Key Paths on the Pi

| Path | Purpose |
|------|---------|
| `/data/umbgs` | Active binary (symlink to umbgs-a or umbgs-b) |
| `/data/config.yaml` | Runtime config (written by dashboard) |
| `/boot/firmware/ground-station.yaml` | Boot config template (read at startup if /data/config.yaml missing) |
| `/data/logs/umbgs.log` | Persistent log file |
| `/etc/systemd/system/umbgs.service` | Systemd unit |
| `/etc/direwolf.conf` | Generated by umbgs at startup |

## Dashboard Architecture

The dashboard is a single HTML file (`internal/dashboard/web/index.html`) embedded via `go:embed`. It uses:
- Vanilla JS (no framework, no build step)
- WebSocket for real-time stats, logs, packets, and config sync
- Dark theme, touch-friendly for 7" Pi screen
- Service worker for offline caching

The `Hub` (`hub.go`) manages WS connections and broadcasts. The `LogAggregator` (`logs.go`) tails `journalctl --output=json` and parses nested slog JSON to extract service names.

## Config System

Config uses optimistic concurrency: GET returns `{version, config}`, PUT accepts `{version, config}` and returns 409 on conflict. Changes broadcast to all WS clients. WiFi PSKs are masked as `"********"` in GET responses; PUT handler preserves real PSKs when masked values are sent back.

All config struct fields have both `yaml:"..."` and `json:"..."` tags — this is critical for the dashboard to work.

## Subsystem Lifecycle

All subsystems follow the pattern: `Run(ctx context.Context) error`. They're started as goroutines in `main.go` and shut down via context cancellation. Direwolf is managed as a subprocess pipeline (`rtl_fm | direwolf`), not a systemd service.

## Important Gotchas

- **Deploy path:** The service runs `/data/umbgs`. Previous sessions wasted time deploying to `/usr/local/bin/umbgs` which did nothing.
- **`go:embed`:** The dashboard HTML is embedded at compile time. You must rebuild the binary to see HTML/CSS/JS changes — there's no hot reload.
- **No tests:** Zero test files exist across all packages. `go vet` passes.
- **Version:** The binary always reports `version: "dev"` unless built with `-ldflags "-X main.version=..."`.
- **Shutdown:** Every stop/restart currently times out at 20s. Subprocess cleanup (cage, direwolf pipeline, journalctl) doesn't complete in time.
- **Pi timezone:** The Pi clock shows BST (+01:00), not UTC. Log timestamps from slog are UTC but journalctl shows local time.
- **Two USB serial devices:** `/dev/ttyUSB0` (claimed by RTL-SDR/direwolf) and `/dev/ttyUSB1` (likely LoRa). The LoRa reader's auto-detect picks the wrong one.
